Prerequisites

  • liar Installed No!

TLDR Checklist

To create some new liar data:

  • Create py file.

  • Import IMakeData and instantiate.

from imakedata.imakedata import IMakeData
number_records = 10
maker = IMakeData(number_records)
  • Create a definition...
datadef = [{
"name": "tldr",
"class": "quicklist",
"data": ["T", "L", "D", "R"]
}]

... or use a ready-made definition.

from imakedata.model.marketing import product_title, postage_cost
from imakedata.model.date import within_days
datadef = [product_title, postage_cost, within_days]
  • Build the data.
my_data_set = maker.get_data(datadef)
print(my_data_set)

Exercise 1: Dicey

Create a new python file in the app where liar is installed. In this example we"re assuming you are putting a file in the eliothing/liar/examples folder called exercise1.py.

Open example1.py.

  • Import the IMakeData module.
from imakedata.imakedata import IMakeData
  • Instantiate the class and set the required number of records.
maker = IMakeData(100)
  • A dice is easy. We can use a simple primitive type:
dice = {
"name": "dice",
"class": "iamprimitive",
"method": "int_list",
"min": 1,
"max": 6,
}

Or a quicklist:

dice = {
  "name": "dice",
  "class": "quicklist",
  "data": [1, 2, 3, 4, 5, 6]
}

A dice is too easy, let's enable a game of backgammon and use two dice. The important thing is not to use the same field name. The utility method is great for reusing definitions and avoiding Passing By Reference gotchas.

  • Import the rewrite_dict utility at the top. If you're a big boy, you don't have to use it.
from imakedata.ijusthelp import rewrite_dict
  • Base your two dice on the first definition, rewriting and copying it as the same time:
dice1 = rewrite_dict(dice, {"name": "dice1"})
dice2 = rewrite_dict(dice, {"name": "dice2"})
  • Now generate the data and print the data to the console:
print(maker.get_data([dice1, dice2]))
  • Finally save and run your script at the shell: python examples/exercise1.py. You should see the following print out.
[   
  {'dice1': 2, 'dice2': 1},
  {'dice1': 5, 'dice2': 5},
  {'dice1': 6, 'dice2': 3},
  {'dice1': 5, 'dice2': 4}
]

We don't know who is playing, so let's add the player of each dice cycle between "white" and "black".

  • Add a toothpaste definition.
player = { "name": "player",
"class": "toothpaste",
"data": ["white", "black"],
}
  • Now generate the data and print the data to the console:
print(maker.get_data([player, dice1, dice2]))

You should now see a printout like this:

[
  {"player": "white", "dice1": 4, "dice2": 1},
  {"player": "black", "dice1": 6, "dice2": 5},
  {"player": "white", "dice1": 6, "dice2": 6},
  {"player": "black", "dice1": 1, "dice2": 5},
  ...
]

Exercise 2: People are tricky

Let's try some a bit useful and put out some data we can use for a contact app.

Create and open example2.py and instantiate IMakeData at the top.

from imakedata.imakedata import IMakeData
from imakedata.ijusthelp import rewrite_dict
number_records = 100
maker = IMakeData(100)

We need some names, and for that we'll use the raw data provided. The raw date comes with first names, genders and titles in one data set, and last names in another.

person = { "name": "person", "class": "igetraw", "data": "person", }
surname = { "name": "surname", "class": "igetraw", "data": "surname", }
print(maker.get_data([person, surname]))

Run this python script and you should You should see something like this:

[
    {
        "person": {"title": "Mr", "first_name": "Felipe", "sex": "male"},
        "surname": "Miriam",
    },
    {
        "person": {"title": "Miss", "first_name": "Ashlee", "sex": "female"},
        "surname": "Fitzhenry",
    },
]

Notice that the "person" field is a JSON object? If you'd prefer a flatter result, add the flatten option.

person = { "name": "person", "class": "igetraw", "data": "person", "flatten": True }
surname = { "name": "surname", "class": "igetraw", "data": "surname" }
print(maker.get_data([person, surname]))

You should see something like this:

[
    {
        "person_first_name": "Portia",
        "person_sex": "female",
        "person_title": "Mrs",
        "surname": "Sevearengen",
    },
    {
        "person_first_name": "Johnathan",
        "person_sex": "male",
        "person_title": "Mr",
        "surname": "Franzmeier",
    },
]

Notice that the field name you chose is joined to the field name of the flattened columns. This is important because it means you can reuse the person raw data and add another person to your data set.

We will extend our data set to produce married couples. For this will use the filters option.

husband = { "name": "husband", "class": "igetraw", "data": "person", "flatten": True, "filters": [{"sex": ["Male"]}], }
wife { "name": "wife", "class": "igetraw", "data": "person", "flatten": True, "filters": [{"sex": ["female"]}, "title": ["Mrs", "Dr"],],  }
surname = { "name": "surname", "class": "igetraw", "data": "surname" }
print(maker.get_data([husband, wife, surname]))

Now our data set has two persons per record, each with unique field names:
