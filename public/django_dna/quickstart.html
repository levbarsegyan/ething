Exercise X: Using elioSin in your Django build

We're using django-livereload-server to make working with an elioSin theme and Django project as easy as possible.

You should already understand the elioSin work-and-build process as described:

  • elioSin prerequisites met? No!
  • god quickstart understood. No!

To run a elioSin theme inside a Django app, follow these steps.

  1. Create a folder inside your Django app called theme (name is optional). Put it alongside your templates folder, directly inside the app folder whose Django templates you are theming. theme is good because it nicelys sits alphabetically below to template.

  2. Use the elioSin generator 'yo sin' to create the boilerplate of a sinful theme in this folder:

    mkdir <theme_name>
cd <theme_name>
yo sin

  3. In your django settings file add the following:

    STATICFILES_DIRS = [os.path.join("<app_name>", "<theme_name>/dist")]
STATIC_ROOT = "<app_name>_static"
STATIC_URL = "/static/"

    elioSin gulp files build resources into the dist folder of the theme, including any image resources like favicon.ico and apple-touch-icon.png.

  4. We're using django-livereload-server to watch+reload the Django Development server - including auto-collecting-static files - when you change any of your elioSin theme files. You should follow the livereload setup described in the link above, but in brief, add these settings to your settings file:

    INSTALLED_APPS = [
"livereload",
...
"django.contrib.staticfiles",
"django_dna.apps.DjangoDnaConfig",
"<app_name>",
]
MIDDLEWARE = [
...
"livereload.middleware.LiveReloadScript",
]

  5. cd into your theme folder and type gulp (which is the standard way you work on an elioSin theme). Now gulp will watch for any changes to your theme's SCSS files and rebuild the CSS files to the dist folder.

  6. In another terminal, cd into your Django project (containing your app) and run: django-admin livereload. This will "watch" for any changes in the STATICFILES_DIRS and collectstatic + reload the Django Development server.

  7. Finally, in yet another terminal, cd into your Django project (containing your app) and run: django-admin runserver (or whatever it is you usually do to start the Django Development server).

NB: You can obviously use this process alongside any CSS framework you are using... but the above steps are 100% the elioWay.
