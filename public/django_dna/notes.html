Notes

In an earlier version, the DJANGO_FIELDS dictionary in dna.py was global to the module. Problem was when a model field type was accessed more than once this creates a issue during migrate and migration.

For instance:

elio_riff.Thing.sameAs: (models.E006) The field 'sameAs' clashes with the field 'sameAs' from model 'elio_riff.thing'.
elio_riff.Thing.sameAs: (models.E006) The field 'sameAs' clashes with the field 'sameAs' from model 'elio_riff.thing'.
elio_riff.Thing.sameAs: (models.E006) The field 'sameAs' clashes with the field 'sameAs' from model 'elio_riff.thing'.
elio_riff.Thing.sameAs: (models.E006) The field 'sameAs' clashes with the field 'sameAs' from model 'elio_riff.thing'.

The explanation for these errors is that 4 fields were using the same instance of models.URLField.

Moving the DJANGO_FIELDS inside the django_type_of method fixed this problem and the whole thing only took about 8 hours to work out.
