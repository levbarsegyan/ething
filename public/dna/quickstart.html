Nutshell

  1. Start your Django Project and install DNA
  2. Define your SITE_DNA
  3. Define some other settings.
  4. Call the DnaReplicator on your models pages.
  5. Define your views.
  6. Define your urls.
  7. Add templates.
  8. Optional: Implement an elioSin style.

With this library you can prototype a new website quickly, as long as you do things the elioWay.

Glossary

  • models and Types and things and classes are the same thing. On schema.org they are called "classes" or "Types" (when they are a property data type). All Types on schema_org are based on the highest class called "Thing". In Django they are represented as Models.

  • fields and properties are the same thing. On schema_org they are called properties; and these are mapped to Django fields of the same name.

Exercise 2 SITE_DNA

The SITE_DNA setting describes the pages you want, which Schema models and fields you intend to use on each page. If you open your settings file you will find clone_dna has already added the following simple example to get you started.

example 1

SITE_DNA = {
  "page": {
    "Home": {
      "engage": {"WebPage": ["name", "description"]},
      "list": {"Photograph": ["name", "image"]},
    },
    "About": {
      "engage": {
        "AboutPage": ["name", "description"],
        "LocalBusiness": ["name", "telephone", "email", "openingHours"],
      }
    },
  }
}

Here is a quick breakdown of the core sections:

page

All the dictionary keys inside a page section will become pages on the website. So in this example there are two pages, "Home" and "About". dna automatically builds a menu and adds it to the context of any page.

engage

There are two "roles" for Schema models per page. The engage role is like a singleton: 1 record of each specified model in this section will be rendered, in order, at the top of the page. For instance in example 1, on the "Home" page, there will be a "WebPage" model whose fields you edit, and are rendered to a template. By default in dna templates, only fields with data are rendered.

list

The second kind of role is for querysets of the Types listed. These are rendered on the page below the engage section. You can list 1 or more Types and they will be listed in order mention in SITE_DNA.

By combining the engage and list sections, you can easily create diverse pages of content.

For example 1 we can expect to see the properties "name" and "description" fields at the top of the webpage, and a list of "images" below. On the "About" page we will see a "name" and "description" the "name", "telephone" and "email" of the site's LocalBusiness below that.

Schema.org Types have lots of properties, so it's a good idea to list the specific fields you want to edit on forms. All fields are available though - and you can use the Browse section of the website.

Let's look at another more complex example:

example 2

SITE_DNA = {
  "page": {
    "Home": {
      "engage": {"WebPage": ["name", "description"]},
      "list": {"Photograph": ["name", "image"]},
    },
    "sep1": {"status": "spacer"},
    "Movies": {
      "engage": {"WebPage": ["name", "description"]},
      "page": {
        "Reviews": {
          "list": {
            "Review": {
              "paginate": 5,
              "template": "movies/reviews.html",
              "field": ["image", "name", "text", "reviewRating"],
            }
          }
        },
        "News": {
          "list": {
            "CreativeWork": {
              "paginate": 10,
              "field": ["image", "name", "text", "creator"]
            }
          }
        }
      }
    }
  }
}

In this example, we've introduced some sub pages, breaking the "Movies" section of the website into two pages.

We're specified a "template" for the "Review" type. We've also indicated that we want to paginate the "Review" and "News" lists.

There is also a separator in the menu.

What we learnt

The SITE_DNA setting is extremely important. The entire website hangs off it. The site's menu is automatically generated against it. The Django models are dynamically migrated and built from it. If you use all the default dna templates, you don't need to do much more. New pages and features can easily be added to the site by simply adding a new section to this dictionary.
