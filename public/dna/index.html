dna

Dropping the pretense than one Thing is unlike another the elioWay

dna is an experiment in expediancy. With this library you can rapidly build Django web apps especially suitable for collecting data, or as a streamlined CMS.

dna drops the pretense that one thing is different from another. Instead of inventing new Models, dna expects you to use any of the hundreds of available Types in microdata sites like Schema.org. Because the application knows what to expect, this enables web developers to almost entire skip the CRUD stage of work. In almost all cases there is already a Model with all the fields you need in Schema.

The obvious advantage of using Schema to model your databases is being able to automatically render your content with prefect microdata.

To create a website then you don't need to create any models or forms. Simply define the shape of your website as a simple JSON dictionary and let dna do the rest. All the models, views, forms and templates will work out of the box. You really can launch as basic website in less than 10 minutes.

The elioWay

dna takes another usual step and encourages you to work the elioWay. elio is an acronym for:

  • engage
  • list
  • iterate
  • optimize

There's nothing unique about this pattern... developers write apps like this all the time. In the elioWay we formalise this as a way of gathering and rendering content. Out of the box, dna is geared toward this approach and the app leads the user to engage with a "Thing", list "Things" related to it, then start the process of engaging with items in list. In this way a rich, semantically connected array of data is iterative built up.

The optimize stage is where you - as a developer - will be more likely to add custom features, and I will write more about that latter.
